# Референсные превью — не часть production-сборки

Файлы в этой папке **не импортируются** основным проектом и не влияют на
сборку сайта. Это законченные визуальные/поведенческие прототипы, созданные
для демонстрации прямо в чате с Claude (используют `window.storage` для
хранения данных, а не реальный Supabase) — чтобы посмотреть и одобрить, как
что-то должно выглядеть, прежде чем переносить в боевой код.

## owner-dashboard-preview.jsx

Панель владельца сервиса: список всех кафе (поиск, статус активно/отключено,
PIN с копированием), добавление нового кафе, редактирование меню (построчно
+ вставка JSON целиком, которая дополняет меню, а не затирает), включение/
отключение, удаление с подтверждением.

**Чего не хватает для боевого использования:**
- Подключения к реальной таблице `restaurants` в Supabase вместо
  `window.storage`
- Отдельного входа/защиты для самой панели (сейчас открыта без пароля)
- Переиспользования `scripts/menu-utils.mjs` для валидации вместо
  захардкоженной копии внутри файла

## team-preview.jsx

Полная модель приложения для официантов и администратора КОНКРЕТНОГО кафе:
выбор имени официанта, меню с гибкими рубриками и поиском, активные заказы
(с таймером и предупреждением после 10 минут, звук/вибрация, настоящие
браузерные уведомления), общая история, роль администратора (вкладки
Активные/Кухня-Бар/Аналитика/История/Стоп-лист/**Меню**), офлайн-режим с
локальной очередью.

Вкладка **"Меню"** у администратора — то, ради чего этот файл сохранён
последним: редактирование рубрик/блюд, защищённое PIN-кодом, который **не
запоминается** и спрашивается заново при каждой попытке зайти в
редактирование (в отличие от обычного входа в кафе). Демо-PIN внутри файла:
`1234` (константа `DEMO_MENU_PIN`).

**Чего не хватает для боевого использования:**
- Все данные (заказы, меню, стоп-лист) сейчас в `useState` на уровне `App` —
  нужно завести настоящие Supabase-запросы (по образцу того, как это уже
  сделано в `../src/App.jsx` для заказов)
- PIN должен сверяться с реальным PIN кафе из Supabase, а не с константой
- Уведомления/офлайн-режим стоит перепроверить на реальном мобильном
  устройстве перед тем, как полагаться на них в проде
