# Промпт для генерации меню (DeepSeek / любой ИИ)

Скопируй текст ниже целиком, замени [НАЗВАНИЕ КАФЕ] и вставь список
блюд/товаров/услуг от клиента в конце — и отправь в DeepSeek (или ChatGPT,
или любой другой чат). Результат сохрани как `incoming/<id-кафе>.json`.

Рубрики могут быть любыми, не только "еда"/"бар" — например «Кальян»,
«Сигареты», «Услуги», «Кондитерская» и что угодно ещё, подходящее конкретному
заведению. ИИ сам предложит подходящие рубрики, если в конце просто описать
заведение и вставить список.

---

ПРОМПТ (копировать всё, что ниже, до конца):

Ты создаёшь JSON-файл меню для сайта приёма заказов официантами. Строго
следуй формату, ничего от себя не добавляй, кроме самих рубрик и блюд.

Формат ответа — ТОЛЬКО этот JSON, без markdown-разметки, без ```json,
без пояснений до или после:

{
  "name": "Название кафе",
  "categories": [
    { "id": "food", "name": "Еда", "icon": "utensils" },
    { "id": "bar", "name": "Бар", "icon": "wine" }
  ],
  "items": [
    { "id": "f1", "name": "Название блюда", "price": 420, "category": "food" },
    { "id": "b1", "name": "Название напитка", "price": 150, "category": "bar" }
  ]
}

Правила, которые нужно соблюдать строго:
1. "name" — название кафе, как есть.
2. Сам придумай рубрики ("categories"), подходящие именно этому заведению,
   на основе списка ниже. Не ограничивайся только "едой" и "баром" — если в
   списке есть кальяны, сигареты, десерты отдельно, какие-то услуги и так
   далее — выдели их в отдельные рубрики с понятными названиями.
3. У каждой рубрики "id" — короткий, латиницей, без пробелов (food, bar,
   hookah, cigarettes, desserts, services и т.п.), не должен повторяться.
4. У каждой рубрики "icon" — выбери ОДНО слово строго из этого списка,
   максимально подходящее по смыслу:
   utensils, wine, flame, cigarette, sparkles, coffee, package, wrench,
   shopping-bag, beer, ice-cream, gift
   Если ничего не подходит идеально — возьми "package".
5. id блюд в "items" — короткие уникальные (латиница/цифры) — главное, чтобы
   НИ ОДИН id не повторялся во всём списке "items".
6. У каждого блюда обязательно поле "category" — id той рубрики, к которой
   оно относится, точно совпадающий с одним из "id" в "categories".
7. "price" — всегда число без кавычек и без "₽"/"руб" (правильно: 420,
   неправильно: "420" или "420 руб").
8. "name" у блюда — только название на русском языке, без лишних описаний.
9. Если в списке ниже не указаны цены — придумай разумные цены в рублях для
   обычного городского заведения такого типа.
10. Не пиши ничего, кроме самого JSON целиком.

Вот список товаров/блюд/услуг заведения «[НАЗВАНИЕ КАФЕ]», раздели их на
подходящие рубрики по правилам выше:

[СЮДА ВСТАВИТЬ СПИСОК ОТ КЛИЕНТА — можно просто скопировать как есть из их
старого меню/прайса, даже неаккуратным текстом, ИИ сам разберётся]

## Что делать с ответом

1. Скопировать ответ ИИ (должен быть чистый JSON с полями name/categories/items)
2. Если в начале/конце всё же приписал ```json и ``` — удалить эти строки
3. Сохранить как `incoming/<id-кафе>.json`, например `incoming/kafe-vostok.json`
   (имя файла = id кафе — латиница, цифры, дефисы, без пробелов)
   - если такого id ещё нет — создастся новое кафе, и скрипт выдаст ему PIN-код
   - если такой id уже есть (клиент просит поменять меню или добавить рубрику)
     — меню обновится, а PIN останется прежним
4. Выполнить: `npm run push-menu`
   — правки применяются в базе данных сразу, ничего пересобирать и пушить на
     GitHub не нужно (сайт один и тот же для всех)
5. Для нового кафе — сообщить клиенту адрес сайта и его PIN-код (скрипт
   выведет его в консоль)

## Если ИИ ошибся в формате

`npm run push-menu` проверяет каждый файл по тем же правилам. Если что-то не
так (цена в кавычках, повторяющийся id, блюдо ссылается на несуществующую
рубрику, нет поля "name") — файл просто не будет обработан, скрипт покажет
точную причину, и можно поправить и запустить снова, не боясь сломать уже
добавленные кафе.
